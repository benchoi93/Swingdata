# Weekly Task: Generate Beamer Slides from Daily Reports

## Overview

Create a weekly **LaTeX Beamer** slide deck (compiled to PDF) summarizing the automated daily progress reports generated by the Claude Code agent (`automation/run_daily.ps1`). This is a **recurring weekly task** — run every Sunday (or end of work week) to compile that week's reports into a single, polished presentation.

**Why Beamer over PPTX:**
- Consistent with existing LaTeX workflow (`src/paper/introduction.tex`)
- Publication-quality typesetting, math support, BibTeX integration
- Version-controllable `.tex` source
- MiKTeX 26.1 already installed with `pdflatex`, `xelatex`, `latexmk`

---

## Inputs

- **Daily reports**: `reports/YYYY-MM-DD.md` (5-7 files per week)
  - Each contains: Session Summary, Tasks Completed, Key Findings, Files Created, Issues/Blockers, Next Session Plan
- **TASKS.md**: Master task list for overall phase tracking
- **Figures**: `figures/*.png` — publication figures produced during the week

## Output

- `reports/weekly/YYYY-Www.tex` — Beamer source (e.g., `2026-W07.tex`)
- `reports/weekly/YYYY-Www.pdf` — Compiled PDF slides

---

## Slide Deck Structure

### Frame 1: Title
- **Title**: "Weekly Progress Report — E-Scooter Speed Safety"
- **Subtitle**: Week range (e.g., "Feb 10--16, 2026")
- **Author**: Prof. Seongjin Choi
- **Institute**: UMN CEGE | TR-C Paper

### Frame 2: Executive Summary
- 3-5 bullet points summarizing the week's highlights
- Synthesized from all daily Session Summary sections
- Number of tasks completed, phases advanced, key milestones

### Frame 3: Phase Progress Overview
- Table or TikZ progress bar: Phase 1-7 with completion status
- Color-coded: green (complete), yellow (in-progress), gray (pending)
- Task counts per phase (e.g., "Phase 5: 8/8")
- Parsed from TASKS.md checkbox counts

### Frame 4--5: Tasks Completed This Week
- Aggregated from all daily reports, grouped by phase
- Each task: ID, brief description, key quantitative result
- Use `itemize` with compact spacing; split across frames if needed

### Frame 6--7: Key Findings
- Top 5-8 most important findings across the week
- Include quantitative results (ORs, R-squared, spatial stats)
- Beamer `tabular` for structured results (model coefficients, city comparisons)
- Math mode for statistical notation ($\text{OR} = 121$, $p < 0.001$)

### Frame 8: New Figures
- `\includegraphics` grid of publication figures created this week
- 2-4 figures per frame with `\subcaption` or `columns` layout
- Pull from `figures/*.png` with creation date in the week range

### Frame 9: Issues & Blockers
- Aggregated from all daily reports
- Two columns: Resolved (left) vs. Still Open (right)
- Methodological decisions made and rationale

### Frame 10: Next Week Plan
- From the most recent daily report's "Next Session Plan"
- Plus pending tasks from TASKS.md
- Prioritized `enumerate` list

## Beamer Theme & Design

```latex
\documentclass[aspectratio=169]{beamer}

% --- Theme ---
\usetheme{metropolis}           % Clean, modern academic theme
\usecolortheme{default}

% --- UMN Colors ---
\definecolor{umnmaroon}{HTML}{7A0019}
\definecolor{umngold}{HTML}{FFCC33}
\setbeamercolor{frametitle}{bg=umnmaroon, fg=white}
\setbeamercolor{title}{fg=umnmaroon}
\setbeamercolor{structure}{fg=umnmaroon}
\setbeamercolor{alerted text}{fg=umngold}

% --- Fonts ---
\usefonttheme{professionalfonts}

% --- Packages ---
\usepackage{booktabs}           % Professional tables
\usepackage{graphicx}           % Figures
\usepackage{subcaption}         % Subfigures
\usepackage{tikz}               % Progress bars, diagrams
\usepackage{xcolor}             % Colors
\usepackage{fontawesome5}       % Icons (optional)
\usepackage{hyperref}
```

### Design Rules
- **Aspect ratio**: 16:9 (`aspectratio=169`)
- **Theme**: Metropolis (clean, modern, widely used in CS/engineering)
- **Colors**: UMN maroon (#7A0019) for structure, gold (#FFCC33) for alerts
- **Max bullets per frame**: 6-8
- **Tables**: `booktabs` style (no vertical rules, `\toprule`/`\midrule`/`\bottomrule`)
- **Figures**: `\includegraphics[width=\linewidth]`, maintain aspect ratio
- **Footer**: Frame number, short title, date range
- **Math**: Full LaTeX math mode for statistics ($\beta$, OR, CI, $p$-values)

## Implementation Plan

### Step 1: Create Beamer template
- `automation/templates/weekly_beamer.tex` — reusable template with UMN branding
- Placeholder tokens like `{{WEEK_RANGE}}`, `{{EXECUTIVE_SUMMARY}}`, `{{TASKS_COMPLETED}}`
- Compiled once to verify theme/packages work

### Step 2: Create the weekly report generator
- `automation/generate_weekly_slides.py`
- **Markdown parser**: Reads `reports/YYYY-MM-DD.md` files for the target week
  - Extracts sections by `## ` headers
  - Converts markdown bullets to LaTeX `\item`
  - Converts markdown tables to LaTeX `tabular`
  - Escapes LaTeX special characters (`%`, `&`, `_`, `#`, `$`, `~`, `^`)
- **TASKS.md parser**: Counts `[x]`/`[ ]` per phase for progress tracking
- **Figure scanner**: Finds `figures/*.png` modified in the week range
- **Template renderer**: Fills Beamer template with extracted content
- **Compiler**: Calls `pdflatex` (or `latexmk`) to produce PDF

### Step 3: Create automation script (optional)
- `automation/run_weekly.ps1` — PowerShell script for Task Scheduler
- Runs every Sunday at 11:00 PM
- Can either:
  - (a) Run `generate_weekly_slides.py` directly (deterministic, fast)
  - (b) Invoke Claude Code to synthesize content (higher quality, variable)

### Step 4: Test with current week's data

## Execution Workflow (Manual — via Claude Code)

When the user says "generate weekly slides" or it's the end of the week:

1. **Identify the week**: Determine date range (Mon--Sun or as specified)
2. **Read daily reports**: Parse all `reports/YYYY-MM-DD.md` in that range
3. **Read TASKS.md**: Get current phase completion status
4. **Scan figures/**: Find figures created during the week
5. **Generate `.tex`**: Write Beamer source with synthesized content
6. **Compile**: `pdflatex` or `latexmk` to produce PDF
7. **Save**: `reports/weekly/YYYY-Www.tex` + `.pdf`
8. **Report**: Show the user what was generated

## Execution Workflow (Automated)

For fully automated weekly runs:

1. Add `automation/run_weekly.ps1` to Windows Task Scheduler
2. Schedule for Sunday 11:00 PM
3. Script runs `automation/generate_weekly_slides.py`
4. Output: `reports/weekly/YYYY-Www.tex` + `.pdf`

## LaTeX Special Character Handling

The Python generator must escape these characters when converting markdown to LaTeX:

| Character | LaTeX Escape | Notes |
|-----------|-------------|-------|
| `%` | `\%` | Common in "98.2%" |
| `&` | `\&` | Table column separators handled separately |
| `_` | `\_` | Common in file paths, variable names |
| `#` | `\#` | Markdown headers stripped before this |
| `$` | `\$` | Unless inside math mode |
| `~` | `\textasciitilde{}` | Or use as non-breaking space |
| `^` | `\textasciicircum{}` | Unless inside math mode |
| `<` | `$<$` | In comparisons like "p < 0.001" |
| `>` | `$>$` | In comparisons like "> 25 km/h" |
