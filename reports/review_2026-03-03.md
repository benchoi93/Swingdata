## Suggested Research Directions

### 1. **Behavioral Substitution After TUB Ban: Risk Compensation Analysis** (Feasibility: HIGH)
   - **Method:** Using the same city-month panel from the DiD analysis, test whether riders compensated for the December 2023 TUB ban through alternative risk-taking behaviors. Specifically, compute city-month averages of (a) mean trip distance, (b) harsh acceleration rate, (c) fraction of trip spent on major roads, and (d) average speed CV (speed variability within trips) for STD/ECO modes only. Apply the same TWFE DiD specification (Equation 7) but with these behavioral outcomes as dependent variables instead of speeding rate.
   - **Data needed:** `data_parquet/all_months/routes_all.parquet` (already available; need to compute city-month aggregates for non-speed behavioral indicators). Speed profile data available from Feb-Dec 2023.
   - **Expected finding:** If risk compensation is present, STD-mode trips should become longer, show higher acceleration rates, or shift to riskier roads after the TUB ban. If no compensation is detected (null result), this strengthens the policy case for mandatory speed governors by ruling out behavioral offsets. Based on the existing finding that within-mode speeding rates are essentially unchanged between newcomers and established riders, we hypothesize minimal risk compensation.
   - **Target venue:** Transportation Research Part C (natural extension of current paper) or Accident Analysis & Prevention (safety-focused).
   - **Connection to current work:** Directly extends the DiD analysis already in the paper. The conclusions already mention behavioral substitution as a future direction ("whether riders compensate for the TUB ban through longer trips, different road choices, or riskier acceleration patterns"). This analysis uses the same identification strategy and panel data, making it a low-cost extension with high policy relevance.

### 2. **Deep Learning on Within-Trip Speed Sequences for Safety Classification** (Feasibility: HIGH)
   - **Method:** Treat each trip's speed profile (variable-length time series of speed readings at ~10s intervals) as input to a 1D-CNN or Transformer encoder for binary classification (speeding vs. non-speeding trip). The key research question is whether the temporal *pattern* of speed changes (acceleration-cruise-deceleration dynamics) is predictive of eventual speeding, beyond the static summary statistics (max speed, mean speed, CV) already used in the logistic regression and LightGBM models. Train on 80% of May 2023 trips, test on 20%. Compare against (a) the existing LightGBM baseline (AUC=0.905) and (b) a simple sequence model using only the first N speed readings (early warning). A particularly novel angle: predict whether a trip will eventually exceed 25 km/h using only the first 30 seconds of speed data.
   - **Data needed:** Raw speed arrays from `data_parquet/routes_filtered.parquet` (the `speeds` column contains per-trip speed sequences). Already available for 2.78M trips. For the 24-month extension, `routes_all.parquet` has the same column.
   - **Expected finding:** The sequence model should match or exceed the LightGBM baseline for full-trip classification. The early-warning model (first 30s) is the novel contribution -- initial acceleration patterns and early cruise speed likely predict eventual speeding. This has direct practical value for real-time geofencing interventions that could throttle a scooter mid-trip. A recent study by [Yurdakul (2025)](https://onlinelibrary.wiley.com/doi/10.1155/atr/8794166) compared ML/DL approaches for e-scooter usage prediction, and [LSTM-based classification of e-scooter trajectories](https://www.tandfonline.com/doi/full/10.1080/15389588.2025.2524466) for tandem riding detection shows the viability of sequence models on scooter data, but no prior work applies sequence classification to speed safety profiles at this scale.
   - **Target venue:** NeurIPS Workshop on ML for Transportation, or Transportation Research Part C (if framed as a practical safety monitoring system).
   - **Connection to current work:** The current paper uses static speed summary statistics (SHAP confirms TUB mode dominates). This follow-up would exploit the *temporal dynamics* within trips that the current analysis explicitly notes as underexplored (the "speed drift" finding: riders slow down -1.5 km/h in the second half of trips). Aligns with PI's expertise in deep generative models for transportation (TrajGAIL).

### 3. **Spatial Spillover Effects: Does Speeding Propagate Across Adjacent Areas?** (Feasibility: MEDIUM)
   - **Method:** Using the H3 hex-level speeding data already computed for 5 cities, estimate a Spatial Durbin Model (SDM) to test whether speeding in one hex predicts speeding in neighboring hexes, controlling for local road infrastructure and demographic composition. The SDM includes both spatially lagged dependent variable ($\rho W Y$) and spatially lagged covariates ($W X$), enabling decomposition of direct and indirect (spillover) effects. Test whether the estimated Global Moran's I values (Seoul 0.213, Daegu 0.712) reflect true spatial contagion or are explained by spatially correlated infrastructure. For a dynamic extension, use the 24-month panel to estimate whether speeding hotspots are *stable* (fixed infrastructure effect) or *migrating* (behavioral contagion).
   - **Data needed:** `data_parquet/spatial/*.parquet` (hex-level aggregates with speeding rates, road infrastructure fractions, cycling infrastructure indicators -- already available for 5 cities). For the temporal extension, would need to compute monthly hex-level aggregates from `routes_all.parquet` (moderate computation, ~2-3 hours).
   - **Expected finding:** Spatial spillover is likely significant but modest: the high Moran's I in Daegu (0.712) suggests strong spatial dependence, but much of it is probably explained by road infrastructure clustering (major road corridors). The SDM decomposition would quantify how much is "true" spillover vs. correlated infrastructure. The dynamic analysis may reveal that hotspots are mostly stable (infrastructure-driven), which would support fixed geofencing rather than dynamic systems. The ITF/OECD [Safer Micromobility report (2024)](https://www.itf-oecd.org/safer-micromobility) emphasizes the need for spatially targeted interventions but lacks the granular spatial econometric evidence this analysis would provide.
   - **Target venue:** Transportation Research Part C or Journal of Transport Geography (spatial focus).
   - **Connection to current work:** The current paper reports Global Moran's I and Gi* hotspot statistics but does not model spatial spillovers or decompose direct/indirect effects. This direction deepens the spatial analysis from descriptive to causal-spatial, directly building on the H3 hex infrastructure already in place.
